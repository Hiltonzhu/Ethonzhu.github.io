“埋点方案漫谈”

27 Dec 2019

前言

  • 本文将为您解析以下开发中埋点相关知识 :
    • 埋点的定义、作用以及收集能力
    • 现行市场上实现埋点方案分类
    • 不同方案各自的利弊
    • 不同方案具体设计、实现以及关键技术

埋点的定义、作用以及收集能力

1、 定义: 数据采集的一种方式 、记录和收集终端用户的操作行为

2、 作用: 数据统计、分析、挖掘、衡量产品指标
    2.1、 驱动决策: ABtest、漏斗优化、bug修复、精准营销、留存/活跃预警、渠道来源推广、投放效果
    2.2、 驱动产品智能: 智能推荐/千人千面、场景化提示/私人助理等
    2.3、 驱动安全: 风险识别

3、  收集能力
    3.1、 事件: 冷启动、页面事件、用户行为、界面滑动相关
    3.2、 业务层: 基于事件、覆盖事件函数相关参数

埋点方案分类

1、 前端
    1.1、 Web、App、小程序: 行为、直接数据相关
        * 手动埋点
        * 无埋点
        * 可视化埋点

2、 后端
    2.1、 服务端: 流程、核心数据

3、 其他
    3.1、 路径 / 独立埋点
    3.2、 显性 / 隐性埋点
    3.3、 业务 / 监测埋点

不同埋点方案各自的利弊

不同方案具体设计、实现以及关键技术

1、 手动 : SDK 植入

2、 无痕: AOP
    2.1、iOS
        * 运行时 Hook
    2.2、Android
        * 静态代理: 打包流程中字节码插装插入/修改类
            * AspectJ: .java -->.class 
            * ASM / Javassist: .class --> .dex 
            * AST: javaCode --> .java
        * 动态代理: 运行时
            * View.OnClickListener
            * Window.Callback
            * View.AccessibilityDelegate

3、 可视化
    * WebSocket + 配置平台
